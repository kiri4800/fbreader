FBReader SDK for Android

The central class of the SDK is TextWidget. The class extends android.view.View and displays a book content. The book can be set by using setBook(Book) method.

Please note that in the current implementation you cannot use two TextWidgets with different books simultaneously. This possibility might be added in future versions of the SDK.

TextWidget class is abstract. You have to extend it and implement two methods:

  • createGestureListener(); please read more about the listener below in User interactions: gestures, etc. section;
  • createController(Book); please read more about the controller below in Book data storage section.

Sample apps and open-source modules

FBReader SDK includes sample apps. We recommend starting from the app simple to understand how to write your app based on the SDK. The app opens an embedded book.epub file. Replace this file with your book file to test how it displays in FBReader SDK. Note that the file extension has no matter: FBReader detects the file type automatically.

The app extensions demonstrates some extra features you can easily add to your app: text search, navigation, table of contents, etc.

FBReader SDK delivered in binary form, but some modules: extras, styles, and text_extras are available in source forms. These modules provide default style configurations and UI for auxiliary features like navigation and selection panels.

The sample apps and the open-source modules can be used in your app with no restrictions. You can include them as is, or modify, or create own implementations for some features. You also can publish this code, modified or not, in any form.

Customisations

The UI made with FBReader SDK is highly customisable. For most features, the library provides a default implementation, taken (sometimes with some simplifications) from the FBReader app. It is possible to change this default behaviour.

The customisable features are listed below, with short instructions about how to do the customisation.

  • Book data storage
  • Colour profiles
  • Default font
  • User interactions: gestures, etc.
  • Visible entity (hyperlink, image, bookmark, etc.) openers
  • Text selection
  • Text search panel
  • Navigation panel
  • Position stack
  • Error messages (toasts)
  • Info view
  • Visited hyperlinks
  • Table of contents
  • Bookmarks

Book data storage

The SDK provides a way to open a book and render its content. However, the SDK does not know how do you store your data between the app runs. Implementation of persistent storage is the app developer responsibility. You have to save and restore the following data:

  • current position per book,
  • bookmarks list per book, and
  • visited hyperlinks info per book.

There is an abstract class TextBookController that you have to implement in your application. To simplify the start of development, we also provide a dummy implementation, AbstractTextBookController that can be used for beginning. However, it is strongly recommended not to use this class in your production app.

The sample app named extensions implements all the methods from TextBookController. Please note that only current position is stored persistently in this sample. As for bookmarks, the sample demonstrates how to serialise and deserialise bookmark to/from string; so you could implement just a string storage.

Again, we do not know how do you want to store large data between runs, so this is your and only your responsibility to implement the persistent storage.

Colour profiles

There are two ways to customise colours used in TextWidget. You can

  1. define a colour profile in resource files, and then select the profile programmatically, or
  2. change a single colour in a defined colour profile from your application code.

We recommend to prefer the way (a) for changing the appearance, and use (b) if you want to provide a way to customise profile to the app users.

The open-source module named fbreader/styles provides four predefined profiles: default light, default dark, eink light, and eink dark. The profiles are defined in fbreader/styles/src/main/assets/profile/*.json files. The default light is selected by default.

To define own colour profile, create profile folder in your app assets, and put <your_profile_name>.json file there. You can then select the profile by calling setColorProfileName(String) with "<your_profile_name>" as argument.

To implement (b), get ColorProfile object via colorProfile() or colorProfile(String) and change the options.

Default font

The font is defined in fbreader/styles/src/main/assets/styles/base.json. You can change this file if you want to use other default values. To change the font dynamically, use BaseStyle object obtained via baseStyle(). Please note that after changing the options you have to call clearTextCaches() and repaint(). See the sample apps for examples.

User interactions: gestures, etc.

Screen touches, swipes, etc. are under the app developer control. You can implement your GestureListener subclass and return an instance via createGestureListener() method. The SDK provides open-source classes GestureListenerBase and GestureListenerImpl that implement the default FBReader behaviour. The sample apps extend GestureListenerImpl.

Visible entity (hyperlink, image, bookmark, etc.) openers

Entity object presents a visible entity on the screen. The entity can be found by coordinates and used during user interaction. In your extension of the TextWidget class, you can register EntityOpeners that describe what to do with entities on user clicks.

Text selection

Selection colour

The colour of text selection stored in selectionBackground.

Selection cursors

You have to override method drawSelectionCursor(Canvas, Point, boolean) do define how to draw the cursors. By default, this method does nothing. To use the FBReader implementation, call drawCursor(BaseWidget, Canvas, Point, boolean).

Selection panel

The selection panel is shown if user selects a text and releases selection cursor. Define showSelectionPanel() and hideSelectionPanel() methods to implement the panel. In FBReader app, the panel is a five-button widget. You can use the implementation as made in extensions sample application. Please note that this implementation expects that the TextWidget has a RelativeLayout parent.

Text search panel

The text search is implemented via search(String) method. We recommend running this method in background mode as the search can take some time.

The method searches the pattern in whole text, and returns an integer, the number of matches. The matches are highlighted in the text. To navigate over the matches use methods hasSearchResults(), findClosest(), canFindNext(), findNext(), canFindPrevious(), and findNext(). To clear the highlightings use method clearSearchResults(). The search is case-insensitive.

You can use open-source FBReader implementation of search navigation panel TextSearchUtil. An example is provided in the extensions sample app.

Navigation panel

The navigation panel is a popup bar, usually in the bottom part of the screen, with a slider that allows navigation over the text. It is not a part of the SDK library. However, to give an idea of how to implement it, fbreader/text_extras module includes an implementation taken from the FBReader app. The extensions sample app provides a usage example.

Position stack

The position stack is a list of positions user has navigated from via "long jumps", i.e. via the table of contents, via the navigation panel, etc. The position stack saves up to three latest positions. Can be used to implement “Return to position” menu like one in FBReader app. In order to support the stack you have to implement bookmarks.

You can access the list of positions (=“invisible bookmarks”) via hiddenBookmarks() method.

Error messages (toasts)

Sometimes TextWidget wants to show to the user an error popup. E.g., if text search fails because no matches found in the text. Override showPopup(int) method to define how to show the messages.

Info view

Info view is a transparent view in the centre of the screen that shows few letters of info for a few seconds. FBReader uses it during the brightness adjustment. To make the view customizable, TextWidget class provides the overridable method showInfo(String). The implementation taken from the FBReader app is available in org.fbreader.extras.info package of fbreader/extras package. Sample usage is presented in the extensions app.

Visited hyperlinks

If you want to display visited hyperlinks in another colour, implement methods markHyperlinkAsVisited(String) and isHyperlinkVisited(String) in your TextBookController.

The extensions sample app provides a non-persistent implementation.

Important note. During the implementation of the persistent storage do not forget that the bookmarks should be stored per book. Methods markHyperlinkAsVisited(String) and isHyperlinkVisited(String) should retrieve only ids for the controller's book, not all ids.

Table of contents

FBReader SDK provides access to the book's table of contents via tableOfContents() method. A sample of usage is presented in the extensions sample app. Please note, that book file can contain or not contain the table of contents. If there is no table of contents in the current book, the method returns null.

Bookmarks

TextWidget support bookmarks of two types: visible and hidden. Visible bookmarks are used as highlightings in the book. Hidden bookmarks are used for saving the positions stack.

In order to support bookmarks you need to implement six methods in the TextBookController: defaultHighlightingStyleId(), highlightingStyles(), saveBookmark(Bookmark), deleteBookmark(Bookmark), iterateVisibleBookmarks(TextBookController.BookmarksConsumer), and hiddenBookmarks(). The extensions sample app provides a sample. Please note that the sample does not store the bookmarks persistently. This means every time you stop the app all bookmarks are lost. To fix this you have to implement persistent storage.

Important note. During the implementation of the persistent storage do not forget that the bookmarks should be stored per book. Methods iterateVisibleBookmarks(TextBookController.BookmarksConsumer) and hiddenBookmarks() should retrieve only bookmarks for the controller's book, not all bookmarks.

If you want to perform some action when the user taps a bookmark, you have to implement an opener for BookmarkEntity class. Do not forget to register the opener in your TextWidget implementation. A sample is provided by the extensions app.
